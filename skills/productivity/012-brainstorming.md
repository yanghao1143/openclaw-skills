# 头脑风暴设计技能

## 概述

头脑风暴技能帮助将原始想法转化为完整的设计方案和规格文档。通过自然的协作对话，逐步提问、探索替代方案、渐进式验证，最终产出可执行的技术设计。

## 适用场景

- 将模糊的想法转化为具体的技术方案
- 在编写代码之前进行设计规划
- 探索多种实现方案并选择最优解
- 创建设计文档和技术规格

## 工作流程

### 1. 理解上下文
- 了解当前项目结构和技术栈
- 明确约束条件和非功能需求
- 确认目标用户和使用场景

### 2. 逐步提问
- 每次只问一个问题
- 从高层次到细节逐步深入
- 提出 2-3 种可选方案供讨论

### 3. 渐进式设计
- 以 200-300 字的小节呈现设计
- 每个小节后确认是否正确
- 根据反馈调整方向

### 4. 输出设计文档
- 完整的技术设计文档
- 数据模型和 API 设计
- 实现计划和任务拆分
